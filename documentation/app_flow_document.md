# App Flow Document

## Onboarding and Sign-In/Sign-Up

When a new visitor arrives at the ai-shop landing page, they are greeted with a clean homepage that highlights featured products and invites them to create an account or sign in. To create a new account, the visitor clicks on the “Sign Up” link in the header. They are presented with a registration form where they can enter their name, email address, and a chosen password. Alternatively, the visitor may choose to sign up using a social login option such as Google, which opens a popup to complete the OAuth process. After submitting the form, the user is either logged in immediately or directed to an email verification page if verification is enabled. If the user already has an account, they can click “Sign In” and enter their email and password or use the social login option again. A “Forgot Password” link sits below the sign-in form; clicking it prompts the user to provide their email address, then sends a reset link. Following the link allows the user to set a new password and return to the sign-in page. Once authenticated, the user’s session is maintained by a secure cookie, and a “Sign Out” option appears in the profile menu to end the session.

## Main Dashboard or Home Page

After signing in, the user lands on the main home page, which features a prominent header with the ai-shop logo on the left, navigation links to Products, Chat Support, and a profile avatar on the right. A shopping cart icon with an item count appears next to the avatar. Below the header, the page displays a grid of product cards showing an image, title, and price for each item. Scrolling down reveals pagination controls or infinite loading to browse additional products. The footer includes links to About, Contact, and Help. For administrators, the header and sidebar include an additional “Admin Dashboard” link that leads to the administrative interface.

## Detailed Feature Flows and Page Transitions

When a user clicks on the Products link or the logo, they are taken to the product listing page at `/products`. This page presents all available items with images and brief descriptions. Clicking a product card navigates to `/products/[id]`, where the user sees detailed information including multiple images, a full description, price, available quantity, and customer reviews. An “Add to Cart” button allows the user to place the item in their shopping cart; clicking it briefly displays a confirmation message and updates the cart count in the header.

Accessing the cart page at `/cart` shows a list of all selected items, each with quantity controls and a remove button. The page also displays a subtotal and a “Proceed to Checkout” button. If the user is not signed in, clicking that button redirects them to the sign-in page. Once signed in, the checkout flow continues on `/checkout`, where the user enters shipping details, selects a shipping method, and provides payment information. After submitting payment details, the system processes the order and redirects the user to `/order/confirmation`, which shows an order number, summary of purchased items, and an estimated delivery date.

The AI chat feature is available on every page via a chat icon in the lower corner. Clicking the icon opens a chat window overlay where the user can type messages. Each message is sent to the `/api/chat` endpoint. The AI responds below the user’s message, and the conversation is stored in the session for review. The chat overlay can be minimized or closed, and reopened without losing conversation history.

Administrators who click the Admin Dashboard link are taken to `/admin`. The admin home page displays analytics widgets showing metrics such as total sales, active users, and current inventory levels. From here, admins can click a “Manage Products” link to view `/admin/products`, a page listing all products with options to add, edit, or delete items. Adding or editing a product opens a form at `/admin/products/new` or `/admin/products/[id]/edit`, where the admin enters product name, description, price, category, and uploads images. After saving, they return to the product list. Similarly, an “Orders” link leads to `/admin/orders`, showing recent orders, their statuses, and details. Clicking an order navigates to `/admin/orders/[id]` to view and update order status. A “Users” link directs to `/admin/users`, where the admin can see registered users and change their roles.

## Settings and Account Management

Users can access account settings by clicking their profile avatar and selecting “Profile.” On `/profile`, they see their personal information such as name and email, which they can change and save. A separate section on this page allows users to update their password by entering their current password, then a new password twice for confirmation. Notification preferences are also found on the profile page, where users toggle options for order updates or promotional emails. After saving changes, the user remains on the profile page, and a success message confirms that their settings have been updated. To return to the main shopping experience, they can click the logo or use the navigation links at the top of the page.

## Error States and Alternate Paths

If a user provides invalid input during sign-up or sign-in, such as an incorrect email format or a password that does not meet requirements, the form displays a clear inline error message next to the affected field. Entering the wrong credentials on sign-in shows a general error above the form. During checkout, if payment is declined or an API error occurs, the page shows a message at the top explaining the issue and offers a button to retry the transaction. If the user loses internet connectivity, a banner appears advising them to check their connection, and any form data remains intact until connectivity returns. Attempting to access admin pages without the proper role triggers a redirect to the main home page with an “Access Denied” notification. Navigating to an undefined route displays a 404 page that explains the page was not found and offers a link back to the homepage.

## Conclusion and Overall App Journey

In summary, a typical user journey begins with arriving on the ai-shop landing page, registering or signing in, and then browsing a catalog of products. The user can explore product details, add items to their cart, and complete a secure checkout to finalize an order. Along the way, the AI chat feature offers assistance and recommendations. Users manage their personal information and notification settings on the profile page. For administrators, a parallel journey involves signing in, accessing the admin dashboard, and performing tasks like managing products, reviewing orders, and viewing analytics. Throughout the app, clear error messages and redirects ensure that users stay on a guided path, making the shopping and management experiences straightforward and reliable.
